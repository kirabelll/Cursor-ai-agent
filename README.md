# AI Powered Development Agent Rules

This directory contains rules and guidelines for AI development agents working in this workspace.

## Overview

The rules defined here establish standards for code quality, security, performance, and development practices to ensure consistent, maintainable, and secure software development.

## Files

### `agent.mdc`
The main rules file containing comprehensive guidelines for AI development agents. This file uses the `.mdc` (Markdown with Cursor) format and is automatically applied to all agent interactions in this workspace.

## Usage

These rules are automatically applied to all AI agent interactions in this workspace through the `alwaysApply: true` configuration in the frontmatter.

### For Developers
- Review these rules before starting development work
- Use them as a checklist during code reviews
- Reference specific sections when making architectural decisions

### For AI Agents
- These rules are automatically loaded and should be followed consistently
- When in doubt, prioritize security and code quality
- Ask for clarification if requirements conflict with these guidelines

## Customization

To modify or extend these rules:

2. Follow the existing structure and format
3. Test changes with sample interactions
4. Document any significant changes in this README
